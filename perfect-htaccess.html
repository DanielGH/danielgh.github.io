The Perfect .HTACCESS setup for passing Google Page Insight

The contents of this .htaccess file have been carefully pieced together to provide answers for common enviromental setups for all websites hosted on servers that support .htaccess configurations. It will work well with the Google Page Insight tool. Here are the sections it addresses:

CROSS-ORIGIN

  • Cross-origin requests (disabled by default)
  • Cross-origin images (enabled: sends the CORS header for images when browsers request it)
  • Cross-origin web fonts (allow for eot|otf|tt|woff2 file types)
  • Cross-origin resource timing (disabled)

ERRORS

  • Error prevention (disable the pattern matching -MultiViews)

INTERNET EXPLORER

  • Document modes (force IE 8/9/10 to render pages in the highest mode available)
  • Iframes cookies (allow cookies to be set from iframes)

SECURITY

  • Clickjacking (example of “X-Frame-Options” response header with “DENY” value)
  • Content Security Policy (CSP) (whitelists trusted sources of content for data on your website)
  • File access (Block access to directories without a default document & Block access to all hidden files and directories; ie. this .htaccess file)
  • HTTP Strict Transport Security (HSTS) (set SSL redirection on the client-side)
  • Reducing MIME type security risks (prevent MIME-sniffing)
  • Reflected Cross-Site Scripting (XSS) attacks (enable filter and prevent browsers from rendering)
  • Disabling Server-side technology information
  • Disabling Server software information
  • Compression
  • Content transformation (disabled)
  • ETags (removed ETags due to using “Expires Headers”)
  • Expires headers

This .htaccess configuration works with static sites and any dynamic ones as well, like WordPress. Append the contents of this file to your working .htaccess file if you are running a website hosted on an Apache websever.

Websites hosted on different HTTP web servers

Not all hosting companies use Apache as their HTTP serving server. Other popular software for running web servers are available like:

  • Internet Information Services (IIS) from Microsoft
  • Nginx (Very powerfull for serving static sites)
  • Lighttpd (Security, speed, compliance, and flexibility)

In case your website is hosted on one of the above HTTP servers and not .htaccess you need not worry, as there are options to convert the configuration of the .htaccess file for them too.

Now you can check the headers of your website using the securityheaders.io online checker tool.
